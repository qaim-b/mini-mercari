# Prompt Pack (Use These As-Is)

## Prompt 1: Product Scope Lock
"Act as a product manager and reduce this feature list to a strict MVP for a 6-week pilot. Keep only features that directly improve listing speed, listing quality, or item discovery success. Output: in-scope, out-of-scope, risks, and measurable acceptance criteria."

## Prompt 2: API Contract Design
"Act as a senior backend architect. Design REST contracts for listing assistance, draft persistence, and feedback capture. Include request/response schema, validation rules, error codes, and latency targets."

## Prompt 3: UX Copy and Flow
"Act as a conversion-focused UX writer. Produce microcopy for listing assistant UI with plain language and high trust. Include empty states, loading states, and error states."

## Prompt 4: Prioritized Engineering Backlog
"Act as a technical program manager. Convert this PRD into P0/P1/P2 tickets with owner, estimate, dependencies, and acceptance criteria. Flag blockers and sequencing risks."

## Prompt 5: Pilot Analytics
"Act as a product analytics lead. Define event taxonomy and dashboard plan to measure listing completion time, suggestion acceptance, conversion lift, and retention."

## Prompt 6: Risk Review
"Act as a staff engineer reviewing production readiness. Identify top operational, model quality, and UX trust risks. Provide concrete mitigations and rollback triggers."

## Prompt 7: Daily Standup Optimizer
"Act as an engineering manager. Given yesterday's progress and blockers, output a daily plan that maximizes delivery against P0 items while protecting quality."

## Prompt 8: Post-Pilot Decision
"Act as a product leadership reviewer. Evaluate pilot results and recommend go/pivot/stop with reasons tied to KPI outcomes and implementation cost."
